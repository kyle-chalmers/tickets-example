# AI Layer Index — tickets-example

A one-line inventory of the Ticketwright AI layer wired into this repo, so humans and agents can
find what exists. Engine (skills, agent, hooks, bin) is provided by the `ticketwright@ticketwright`
plugin; repo-specific config lives here. Generated by `/ticketwright:setup` on 2026-07-04.

## Front-door skills (invoke with `/ticketwright:<name>`)
- **ticket** — open or resume a ticket; auto-loads its context + prior art and routes to the next step.
- **spec-and-build** — write a research-rich spec (committed first), then implement it in fresh context.
- **review** — independent QC pass over a ticket's deliverables → APPROVE / REQUEST-CHANGES.
- **ship** — gated delivery; hard-halts before any external post (tracker/chat/docstore/PR).
- **productize** — turn recurring ticket work into a reusable, golden-tested skill.
- **refresh** — maintain the ticket catalog (`index`) and the domain knowledge pack (`context`).
- **setup** — configure the repo, add a seam (`/setup chat`), or onboard a teammate (`/setup --teammate`).

## Agent
- **qc-reviewer** — independent, read-only second-context reviewer spawned by `/review` / before ship.

## Hooks (wired by the plugin manifest; run from `${CLAUDE_PLUGIN_ROOT}`)
- **PreToolUse** `db_write_guard.py` — asks before any destructive warehouse statement (even in a `-f` file).
- **SessionStart** `session_context.py` + `ticket_index_context.py` — primes each session with the stack + ticket catalog.
- **PostToolUse** `regenerate_ticket_index.py` — keeps `tickets/INDEX.md` fresh on folder changes.

## Adapters in use (vendored into this repo)
- `adapters/tracker/jira.md` — Jira via `acli` + Atlassian MCP.
- `adapters/warehouse/bigquery.md` — BigQuery via `bq`.
- `adapters/vcs/github.md` — git + `gh`.
- (chat + docstore adapters ship with the plugin; add a seam with `/ticketwright:setup <seam>`.)

## Config + catalog
- `.claude/config/stack.yaml` — the single source of truth (seams → tools, project facts, 9 policies).
- `tickets/index_data.json` → `tickets/INDEX.md` (+ `OBJECTS.md`) — the self-maintaining ticket catalog.
- `AGENTS.md` — the always-loaded global rules tier.
